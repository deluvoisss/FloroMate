# FloroMate

FloroMate — интеллектуальная платформа для распознавания растений на фото, ведения энциклопедии и взаимодействия с AI-ботом по уходу за растениями.

## Структура репозитория

```
floromate/
│
├── server/
│   ├── server-plantnet.js    # Сервер для PlantNet (распознавание растений)
│   └── server-gigachat.js    # Сервер для GigaChat AI (чат-бот)
├── src/                      # Фронтенд (React/TypeScript)
├── public/
├── package.json
└── README.md
```

---

## Быстрый старт

### 1. Установи все зависимости

```bash
npm install
```

---

### 2. Отредактируй переменные в серверных файлах

- `server/server-plantnet.js` — проверь прокси и API ключ PlantNet.
- `server/server-gigachat.js` — укажи актуальные адрес и ключ GigaChat AI.
  - Если нет настоящего AI-API, оставь временную заглушку (см. комментарии в коде).

---

### 3. Запуск серверов и фронтенда

#### Только оба сервера:

```bash
npm start
```

#### Только фронтенд:

```bash
npm run start:frontend
```

#### Всё вместе (оба сервера + фронт):

```bash
npm run start:all
```

---

### 4. Использование

- Загружай фотографии растений через интерфейс (Plant Recognition).
- Общайся с AI-ботом по уходу за растениями (ChatAssistant).
- Пользуйся энциклопедией с фильтрацией и карточками растений.

---

### 5. Важно

- Для корректной работы PlantNet сервер использует зарубежный прокси (настраивается в `server-plantnet.js`).
- Сервер AI (GigaChat) не использует прокси.
- Проверь локальные адреса:
  - PlantNet API: `http://localhost:3001/identify`
  - GigaChat API: `http://localhost:3002/api/chat`
- Все настройки портов вынесены в серверные файлы.

## Сборка и деплой

Для продакшн-сборки фронтенда:
```bash
npm run build
```
Результат будет в папке `build/`.

---

## Кастомизация

- Для новых прокси/API-ключей редактируй соответствующие серверные файлы в папке `server`.
- Для изменения внешнего вида — меняй файлы в `src/pages`, `src/App.tsx` и соответствующие CSS.

---

## Решение проблем

- Если чат не отвечает: проверь адрес AI API и ключ, либо используй заглушку.
- Если PlantNet не работает — проверь настройки прокси и работоспособность вашего прокси-сервера.
- Для модульного запуска открой параллельно два терминала и выполни:
  ```bash
  npm run start:plantnet
  ```
  и
  ```bash
  npm run start:gigachat
  ```
